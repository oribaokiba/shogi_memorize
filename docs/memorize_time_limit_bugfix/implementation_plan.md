# 定跡暗記持ち時間機能のバグ修正計画

定跡暗記の持ち時間機能における以下の不具合を修正します。

1. **時間切れ時に挙動が不安定になる不具合**
   - 相手（CPU）の着手待ち（`sleep` による遅延）の最中に時間切れが発生すると、タイマー側の強制進行処理と、`sleep` から復帰した自動進行処理が競合し、インデックスエラーや進行バグが発生していました。
   - 時間切れ処理と非同期の自動進行処理が競合しないようにガード処理を追加します。
   - `handleTimeUp()` と Vue コンポーネント側での二重ダイアログ呼び出しを整理します。

2. **相手（CPU）の着手時にもフィッシャー（インクリメント）時間が増加する不具合**
   - プレイヤーが指した手（正解手）のときのみフィッシャー時間を加算するようにし、相手が自動で指した手に対しては持ち時間の加算を行わないよう制御します。

3. **ユーザー名が「▲▲先手」のように重複する不具合**
   - `BoardView.vue` 内部で自動的に `☗`/`☖` が付与されるため、`BoardPane.vue` 側からプレフィックス（手番マーク）を除去し、単に `"あなた"`/`"相手"` を返すようにします。

4. **相手の手番待ちの間もプレイヤーの持ち時間が消費される不具合**
   - プレイヤーの着手後、次の手が相手の手番であれば、フィッシャー加算などを行った上で一旦タイマーを一時停止（`stopTimer`）させます。相手が指した直後に、再度プレイヤー側のタイマーを始動させます。

5. **全体持ち時間設定時に相手側にも同じ残り時間が設定される不具合、および指した瞬間しか表示が更新されない不具合**
   - `store/index.ts` のタイマー表示更新処理で、全体持ち時間モードであってもプレイヤーの色の時計にのみ残り時間を表示するようにします。
   - `memorize.ts` の `totalTimeMs` ゲッターで、タイマー稼働中の経過時間（`getCurrentIntervalMs()`）を正しくマイナスしてリアルタイムにカウントダウンが更新されるようにします。

6. **全体持ち時間モードでフィッシャー（インクリメント）時間および秒読みが正常に機能しない不具合**
   - クラスに全体のフィッシャー加算累積時間 `_totalFisherMs` を追加し、着手ごとに加算します。
   - 全体残り時間（`totalTimeMs`）が残っている間は秒読みを非表示にし、全体残り時間が 0 になった後に初めて秒読みのカウントダウンが行われるように制御します。また全体残り時間があるうちは、秒読みを毎手リセットし続けることで秒読みが消費されないようにします。

7. **持ち時間設定ダイアログのガード不足の不具合**
   - 持ち時間・秒読み・加算の総計値が 0 の状態では、解答スタートボタンを押せないように制限を追加します。

8. **時間切れ時に不快な駒音が連続して鳴る不具合**
   - `handleTimeUp()` および `handleTotalTimeUp()` における最終局面への同期強制進行処理から、`playPieceBeat` (駒音の再生) を除去します。
