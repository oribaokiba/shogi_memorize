# バグ修正ウォークスルー

定跡暗記の持ち時間機能に関して、以下の問題を修正しました。

## 修正内容

### 1. 時間切れ時に挙動が不安定になる不具合の修正
- **問題**: 相手の手番の自動進行待ち（`sleep`）の最中に時間切れが発生すると、タイマーによる自動進行処理と競合してインデックスエラー等が発生していました。
- **対処**: `doMemorizeMove` の `sleep` 直後に `this._isGiveUp` やステップ数の境界ガードを追加し、時間切れ発生時には非同期の自動進行処理が途中で終了するようにしました。また、`handleTimeUp` 時に自動進行処理を明示的にキャンセル（`_isMemorizeProcessing = false`）し、二重で結果ダイアログの表示処理が走る問題を防ぎました。

### 2. 相手の手番でフィッシャー時間が増加する不具合の修正
- **問題**: 相手（CPU）が自動で手を指す際にも、タイマーの再スタート時にフィッシャー時間が加算されていました。
- **対処**: `advanceClockAfterMove` に `isPlayerMove` 引数を導入し、プレイヤーの着手（正解手の入力）時のみフィッシャー時間を加算するように限定しました。

### 3. ユーザー名が「▲▲先手」のように重複する不具合の修正
- **問題**: `BoardView.vue` のテンプレート側で既に `☗` や `☖` が自動付与されるため、`BoardPane.vue` で文字列に付与していたプレフィックスが重複していました。
- **対処**: `BoardPane.vue` 側からは手番マークを除去し、単に `"あなた"` / `"相手"` と返すようにしました。

### 4. 相手の手番待ちの間もプレイヤーの持ち時間が消費される不具合の修正
- **問題**: プレイヤーが正しい手を指した直後、相手が指すまでの間もタイマーが動き続けていました。
- **対処**: 次の手が相手（CPU）の手番である場合は、着手後にタイマーを一時停止（`stopTimerAndApplyIncrement`）させ、相手が指した後に再開するよう制御しました。

### 5. 全体持ち時間設定時に相手側に同じ持ち時間が設定され、また指した瞬間しか表示が更新されない不具合の修正
- **問題**: 全体持ち時間（`total` モード）の際、両対局者の時計に同じ値が設定されており、さらにタイマー稼働中の現在の経過時間がマイナスされていなかったため、指した瞬間しか値が更新されませんでした。
- **対処**:
  - `store/index.ts` のタイマー更新処理にて、`total` モードでもプレイヤー側の時計のみに残り時間を表示するよう修正しました。
  - `memorize.ts` の `totalTimeMs` ゲッターに `getCurrentIntervalMs()` を考慮した経過時間の計算を追加し、リアルタイムにカウントダウンが更新されるようにしました。

### 6. 全体持ち時間モードでフィッシャー（インクリメント）時間および秒読みが正常に機能しない不具合の修正
- **問題**: 全体持ち時間モード時の累積フィッシャー加算時間が正しく管理されておらず、また秒読みが毎手指すたびに開始・リセットされる秒読みのような挙動になっていました。
- **対処**:
  - クラスに全体のフィッシャー加算累積時間を示す `_totalFisherMs` を導入し、着手ごとに加算させ、ベースの残り時間計算に反映させました。
  - 全体残り時間（`totalTimeMs`）が残っている間は秒読みを非表示（画面上は `-1`）にし、全体残り時間が 0 になった後に初めて秒読みのカウントダウンが始まるよう制御しました。また全体残り時間があるうちは、秒読みを毎手リセットし続けることで秒読みが消費されないようにしました。

### 7. 持ち時間設定ダイアログのガード不足の不具合の修正
- **問題**: 「持ち時間を使用する」設定にした際、持ち時間・秒読み・加算時間がすべて 0 である場合でも解答を開始できてしまい、開始即時間切れになっていました。
- **対処**: `MemorizeSolveDialog.vue` の `canStart` の条件に、持ち時間・秒読み・加算時間の合計値が 0 より大きいことという条件を追加し、すべて 0 の場合は解答スタートボタンが有効化されないようにガードしました。

### 8. 時間切れ時に駒音が連続して（先走って）鳴る不具合の修正
- **問題**: 時間切れ時に残り手番を最終局面まで同期的に強制進行させるループにおいて、毎回 `playPieceBeat` が呼ばれて不快な連続音が鳴っていました。
- **対処**: `handleTimeUp` および `handleTotalTimeUp` 内の強制進行ループから `playPieceBeat` の再生処理を削除しました。
