# 移植先プロジェクトの将棋盤機能の再構成計画

新プロジェクト `new_shogi_memory` において、本来の暗記機能が正しく動作するように、将棋盤を描画する `BoardView.vue` に不足している重要機能を移植・再構成します。

## 解決すべき問題と対応策

1. **成・不成の選択機能の欠落**
   - 駒が敵陣（先手なら1〜3段目、後手なら7〜9段目）に入ったとき、または敵陣から出る・敵陣内で動くときに、「成」「不成」を選択できるダイアログを実装します。
   - ただし、歩や香が1段目（または香なら1段目、桂なら1・2段目）に行くなど、「成る」ことが義務付けられている（不成が反則となる）場合は、ダイアログを出さずに自動で成るようにします。

2. **持ち駒を打つ（Drop）機能の欠落**
   - 駒台の領域のクリックを正しく判定し、持ち駒が選択された状態を作れるようにします。
   - 持ち駒が選択されている状態で盤上の空いているマスをクリックした際、`from` を `null` とした Drop手（持ち駒を打つ手）の `Move` インスタンスを生成して暗記ロジックに送信します。

## 予定する変更内容

### [Component] 将棋盤 (BoardView.vue)

#### [MODIFY] [BoardView.vue](file:///c:/Users/khnhb/Downloads/game/shogi_memory/new_shogi_memory/src/components/BoardView.vue)
- **成・不成の制御**:
  - `promotionPrompt` というリアクティブな状態を追加し、成るかどうかの選択肢を画面上に HTML/CSS でオーバーレイ表示します。
  - 移動が成立した際、成れる条件（`from` または `to` が敵陣に入り、かつ成れる種類の駒である場合）を満たしていればプロンプトを表示します。
  - 不成が反則になる場合（歩・香が最奥段、桂が最奥2段）は、自動で `promote = true` として処理します。
- **持ち駒の選択と打つ処理**:
  - キャンバス内の駒台クリック判定を実装し、持ち駒がクリックされたら `selectedHandPiece` 状態に保持し、駒台上の選択駒を視覚的にハイライトします。
  - `selectedHandPiece` が選ばれている状態で盤上のマスをクリックした際、`new Move(null, sq, false, ...)` を作成して `move` イベントを発火します。
