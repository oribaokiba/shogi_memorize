📝 要件定義書：将棋定跡暗記アプリ（ShogiHomeベース・ANKIFクローン）
1. プロジェクト概要・目的
本プロジェクトは、オープンソースの将棋GUIアプリ ShogiHome（Vue 3 / TypeScript / Vite）[1] をベースに、特定の棋譜（KIFファイル）[2] を繰り返し練習して暗記するための**「定跡暗記アプリ（ANKIF [1]風）」**をWebアプリとして開発することを目的とします。
不要な機能（AI検討、対局、解析など）[3] を大胆に削ぎ落とし、シンプルかつ手軽に定跡クイズが行える環境を構築します。
2. システム構成・ベース技術
ベースコード: ShogiHome（ブラウザ/Webアプリ版）[4]
言語・フレームワーク: TypeScript, Vue 3, Vite[1]
ルールエンジン / 盤面ロジック: ShogiHome既存の盤面GUI・駒操作ロジックを活用（新規の将棋ルール判定エンジンは実装せず、指し手の文字列一致による簡易判定を採用）。
3. スコープ（実装・削除機能の整理）
❌ 削除・非表示にする機能（既存の不要機能）
以下の機能は画面から排除、またはコンポーネントを無効化します[3]：
USIエンジン連携・設定機能（エンジン追加、エンジン初期設定など）[4][5]
検討・解析・詰み探索機能[3]
通信対局・CSAサーバー接続機能[6]
不要な設定項目（思考エンジン関連のタブや設定、評価値グラフ表示など）[3]
盤面の「局面編集モード」（暗記モードに集中させるため）[3]
🟢 残す機能（流用する機能）
将棋盤・駒の描画機能、ドラッグ＆ドロップ（またはタップ）による駒操作[7]
KIFファイルの読み込み（ドラッグ＆ドロップ / ファイルインポート）[8]
盤面の反転機能[3]
4. 新規追加・改修機能要件
① KIFファイルパースと「問題（パス）」の自動生成
KIFファイルをインポートした際、分岐（変化）を含めて自動的に独立した「問題」としてバラします[9]。
仕様:
インポートされたKIFファイルの棋譜ツリー（メインの指し手 ＋ 変化手順）を走査します[9]。
ルート（開始局面）から各終端（葉ノード）までのパスをそれぞれ1つの「独立した問題（定跡ルート）」として抽出します。
各パスに対して自動的に問題名を付与します（例: [KIF名] - 本譜, [KIF名] - 〇手目変化）。
抽出した問題の一覧を画面の「問題選択リスト」に表示します。
② 暗記モード（定跡クイズ）の挙動仕様
選択した問題（指し手シーケンス）に対して、クイズを実行します。
手番の自動判定:
問題の各手番において、「プレイヤーの手番」か「CPUの手番」かを判定します。
初期設定（例：先手番の問題の場合、奇数手がプレイヤー、偶数手がCPUとなります）。
プレイヤーの手番（クイズ状態）:
プレイヤーは盤上の駒を動かします。
判定ロジック（簡易判定）[3]:
プレイヤーが指した手（移動元・移動先・成/不成）が、現在設定されている問題（パス）の「正しい次の手」と完全一致（USI表記や内部オブジェクトでの一致）するかを検証します。
正解の場合: そのまま盤面に指し手を反映し、自動で次の「CPUの手番」に移行します。
不正解の場合:
移動した駒を即座に「移動前の位置」に強制ロールバック（巻き戻し）します。
盤面を赤く一瞬フラッシュさせる、またはエラー音を鳴らすなどの「間違えた視覚・聴覚エフェクト」を返します。
手数は進めず、プレイヤーが正解を指すまでやり直させます。
CPUの手番（自動進行）:
プレイヤーが正解を指した後、短時間（例：500ミリ秒のウェイト）を置いて、問題に登録されている「正しいCPUの手」を自動的に盤面に反映します[10]。
自動進行後、再びプレイヤーの手番に戻ります。
③ 暗記モード用の簡易UI追加
画面のサイドバー、または棋譜一覧エリアを改修し、以下のコントロールを配置します[3]：
KIFインポートエリア（ドラッグ＆ドロップ対応）[8]
問題リスト（変化分岐リスト）: 分割された各パスを選択するボタン。
暗記モード操作ボタン: 「最初からやり直す」「ギブアップ（正解の手を1手表示する）」
プログレス表示: 「現在：10手目 / 全24手」のような進捗表示。
5. 開発時のステップ（AIへの開発指示ガイド）
ステップ1：不要機能の掃除
まず不要なメニュー項目（USI設定、エンジン管理、解析機能など）[3][5] をUI上から見えなくするか、コンポーネントコード自体を整理・コメントアウトしてください。
ステップ2：KIF分岐パース関数の作成
ShogiHomeの既存のKIFパーサーまたは棋譜データストアを使い、棋譜ツリーから「開始局〜本譜末尾」「開始局〜変化分岐末尾」までのフラットな配列（Move[]）を生成する処理を書いてください[9]。
ステップ3：指し手検証・ロールバックの実装
盤面で駒が動いた際のイベントハンドラ（onMove 等）をインターセプトします。
暗記モード中 && プレイヤー手番 の場合のみ、指された手が 問題[現在手数] と一致するかをチェックし、不一致なら盤面状態を強制的に一つ前の状態（または undo）に戻す処理を実装してください。
ステップ4：CPU手番の自動実行
プレイヤーが正解を指した直後、setTimeout 等で微小な遅延を挟み、次の手を自動実行するロジックを統合してください。
